Interface SudoSecureVaultClient

Client responsible for interacting with Secure Vault service to manage highly sensitive user data such as login credentials. Most APIs require two-factor authentication, and data is encrypted and decrypted on the client. Before creating a new vault, a new Secure Vault service user must be registered via register API. The cryptographic key and the password registered are then required in subsequent API invocations. Note: The sensitive input data such as key and password should be zeroed after using them to call the API provided by this client.
